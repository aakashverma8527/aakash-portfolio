📘 Project 1: eCommerce Product Data Validation SOP

A structured SOP to validate product listings across Walmart and Amazon by comparing attributes such as brand, model, size, color, and condition.

🧩 Process Steps
  1. Open Walmart and Amazon product pages.
  2. Compare key attributes side-by-side.
  3. Decide match type based on logic below.
  4. Record decisions in the tracking sheet.
📊 Match Type Logic Table
Match TypeDefinitionCriteria
Exact MatchIdentical attributesBrand, model, size, color, and condition fully match
Close MatchMinor variationSame model, slight brand/color difference
No MatchMajor differenceDifferent product or specs mismatch
📋 Attribute Comparison Rules
  • Fabric: “100% Cotton” ≠ “Cotton Blend” → No Match
  • Condition: New vs Used grades must align
  • ISBN/UPC: Must match if present
  • Color: Accept minor naming variation with image match
  • Size: Both numeric and label sizing must match
📚 Comic Book Grading
GradeAbbreviationDescription

🚫 Out-of-Stock & Ignored Attributes
  • Out-of-stock: Use cached images/info to verify
  • Ignored: Seller name, shipping time, rating

🏫 Project 2: User-Submitted Institution Validation Workflow

Managed continuous review of user-submitted institutions by verifying legitimacy, metadata extraction, and classification for database inclusion.

🔍 Validation Workflow
  • Match institution name with official websites (via 4icu.org, Google Maps, Wikipedia)
  • Classify as: To Add, To Merge, To Reject
  • Extract full metadata: address, population, acronym, coordinates, website, etc.
🧠 Decision Logic
StatusCriteria
To AddUnique valid institution with full metadata
To MergeDuplicate entry under alternate name or spelling
To RejectUnverifiable, fake, or non-educational submission

📌 Research Tips Used

  • Use 4icu.org for rank & legitimacy
  • Confirm using Google Maps & street view
  • Check regional Wikipedia articles (EN + local language)
